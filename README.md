# Тестовое задание - База пользователей

Проект реализован с использованием стека технологий согласно ТЗ.

## Стек технологий

- TypeScript
- Webpack + Npm
- React 16
- Ant Design 5 и styled-components
- React Router 6
- TanStack Query 4 + axios
- dayjs
- Архитектура FSD (Feature-Sliced Design)

## Установка и запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите проект в режиме разработки:
```bash
npm run dev
```

3. Откройте браузер по адресу: http://localhost:3000

## Сборка проекта

```bash
npm run build
```

## Структура проекта

Проект организован по архитектуре Feature-Sliced Design:

```
src/
├── app/              # Инициализация приложения
│   └── providers/    # Провайдеры (роутер, query client)
├── pages/            # Страницы приложения
│   ├── login/        # Страница авторизации
│   ├── users/        # Страница списка пользователей
│   └── not-found/    # Страница 404
├── widgets/          # Композитные блоки
│   └── user-modal/   # Модальное окно создания/редактирования пользователя
├── features/         # Функциональность (пока не используется)
├── entities/         # Бизнес-сущности (пока не используется)
└── shared/           # Переиспользуемые компоненты и утилиты
    ├── api/          # API запросы
    └── lib/          # Хуки и утилиты
```

## Функциональность

### Страница авторизации (/login)
- Логин: `admin`, Пароль: `admin`
- Промис с таймаутом 2000мс
- Сохранение токена в localStorage
- Редирект авторизованных пользователей на страницу пользователей

### Страница списка пользователей (/users)
- Отображение списка пользователей в таблице
- Форматирование даты через dayjs (DD.MM.YYYY)
- Кнопка создания пользователя
- Клик на аватар или имя для редактирования
- Кнопка выхода

### Модальное окно создания/редактирования пользователя
- Валидация полей (обязательные поля, корректность URL)
- Блокировка кнопок и закрытия модального окна во время запроса
- Поле ID отключено при редактировании

### Страница 404
- Использует компонент Result из Ant Design

## API

В проекте используются заглушки для API запросов. Для подключения реального API из mockapi.io необходимо:

1. Создать ресурс на https://mockapi.io
2. Обновить базовый URL в `src/shared/api/axios.ts`
3. Раскомментировать реальные запросы в `src/shared/api/users.ts`

## Авторизация

Токен авторизации сохраняется в localStorage. При обновлении страницы проверяется наличие токена для определения авторизованного пользователя.
